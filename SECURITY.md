# 安全策略

## 支持范围

当前安全维护覆盖 `main` 分支和最新发布版本。历史实验分支、未发布的本地运行数据、第三方平台风控变化和用户自定义部署环境不在安全支持范围内。

## 漏洞报告

请不要通过公开 issue、讨论区或社交媒体披露尚未修复的漏洞细节。报告安全问题时，请使用仓库启用的 GitHub Private Vulnerability Reporting；如果该功能暂未启用，请联系维护者提供的私有安全邮箱或私有渠道。

报告中建议包含：

- 受影响版本、提交或部署方式。
- 可复现步骤和最小化请求样例。
- 影响范围，例如认证、CSRF、文件 token、Stripe webhook、AI 配置、临时文件、依赖供应链或下载边界。
- 是否需要第三方账号、真实支付环境、AI API key、Cookie 或公开视频平台登录态。

维护者会优先确认报告是否可复现，再评估影响范围并安排修复。高危问题在修复前不应公开传播利用细节。

## 安全边界

SaveAny 面向本地或自托管场景，仅处理用户有权访问的公开视频和用户自行配置的 AI、Stripe、站长平台服务。项目不提供 DRM 绕过、付费内容绕过、私密内容抓取、用户 Cookie 托管、共享账号、平台登录态代管或验证码绕过能力。

生产部署时，维护者需要自行配置 HTTPS、受限 CORS、强随机盐、生产 Cookie 安全属性、Stripe webhook secret、AI API key、日志保留策略、运行时文件清理、依赖漏洞治理和滥用防护。

## 密钥处理

不要提交 `.env`、真实 API key、Stripe secret、webhook secret、站长验证 token、Cookie 文件、数据库、日志、下载文件、转写音频或临时运行目录。开源仓库只应保留 `.env.example` 中的占位符。
