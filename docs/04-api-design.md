# API Design

## `GET /api/health`

Returns service health.

```json
{
  "status": "ok",
  "service": "free-video-downloader"
}
```

## `POST /api/analyze`

Form request:

- `url`: required string.
- `cookies_file`: optional `cookies.txt` upload.

Response:

```json
{
  "kind": "video",
  "id": "abc123",
  "title": "Example title",
  "webpage_url": "https://example.com/watch",
  "thumbnail": "/api/proxy/assets/asset-token",
  "duration": 120,
  "extractor": "youtube",
  "formats": [],
  "subtitles": [],
  "entries": []
}
```

For playlists, `kind` is `playlist` and `entries` contains selectable video entries.
Remote thumbnails are rewritten to local proxy URLs so the browser does not directly request hotlink-protected media hosts.

## `POST /api/download`

JSON request:

```json
{
  "url": "https://example.com/watch",
  "entry_ids": ["abc123"],
  "format_id": "best",
  "subtitle_langs": ["en"],
  "write_auto_subs": false,
  "prefer_srt": true,
  "cookie_ref": "optional-task-cookie-ref"
}
```

Response:

```json
{
  "task_id": "task_123"
}
```

## `GET /api/tasks/{task_id}`

Returns a task snapshot:

```json
{
  "id": "task_123",
  "status": "downloading",
  "progress": 42.5,
  "message": "Downloading",
  "download_url": null,
  "error": null
}
```

## `GET /api/tasks/{task_id}/events`

Server-sent events stream with `text/event-stream`.

Event payload:

```json
{
  "status": "downloading",
  "progress": 42.5,
  "message": "Downloading",
  "download_url": null,
  "error": null
}
```

## `GET /api/proxy/assets/{token}`

Returns a remote media asset registered during analysis, primarily thumbnails. The backend requests the upstream asset with the source page as `Referer`, then streams it back to the browser.

Rules:

- Tokens are generated by the backend; callers cannot pass arbitrary upstream URLs.
- Unknown tokens return 404.
- Upstream failures return 502.
- Responses include `Cache-Control: private, max-age=3600`.

## `GET /files/{token}`

Returns the completed file for a known token. Unknown tokens return 404.
